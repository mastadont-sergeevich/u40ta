# Модуль email

Система приёма, обработки и хранения вложений из электронных писем через IMAP и SMTP. Модуль автоматически проверяет почтовый ящик, извлекает вложения (Excel-файлы), анализирует их структуру, сохраняет на диск и в базу данных, а также отправляет уведомления отправителю о результате обработки.

## Сущность EmailAttachment (`email-attachment.entity.ts`)

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | `bigint` | Первичный ключ, автоинкремент |
| `filename` | `varchar(500)` | Имя файла на диске (уникализированное) |
| `email_from` | `varchar(500)` | Адрес отправителя |
| `received_at` | `timestamptz` | Время получения, по умолчанию `NOW()` |
| `doc_type` | `varchar(10)` | Тип документа: `'ОСВ'` (оборотно-сальдовая ведомость) или `'ОС'` (основные средства) |

| `is_inventory` | `boolean` | Признак инвентаризационной ведомости (определяется по теме письма) |

## API Endpoints
**Защита**: JwtAuthGuard

### POST `/api/email/check`
Инициирует ручную проверку почтового ящика. Запускает процесс загрузки новых писем, обработки вложений и отправки уведомлений.
**Ответ**:
{
  "success": true,
  "message": "Проверка почты завершена"
}

### GET /api/email/attachments
Возвращает список всех email-вложений, доступных текущему пользователю. Фильтрация выполняется по складам, к которым у пользователя есть доступ через таблицу mol_access.
**Ответ**: массив EmailAttachmentResponseDto

### DELETE /api/email/attachments/:id
Удаляет вложение по ID. Выполняет проверку прав доступа (пользователь должен иметь доступ к складу вложения), удаляет физический файл с диска и запись из базы данных. После успешного удаления отправляет SSE-уведомление.
Параметры пути: id — числовой идентификатор вложения
**Ответ**: DeleteAttachmentResponseDto


## Обработка вложений

### Поддерживаемые типы документов

**ОСВ** (оборотно-сальдовая ведомость)
Обязательные колонки: Завод, Склад, КрТекстМатериала, Материал, Партия, Запас на конец периода

Параметры:
docType = 'ОСВ'
zavod — из колонки Завод (приводится к числу)
sklad — из колонки Склад (первое непустое значение)

**ОС** (основные средства)
Обязательные колонки: Основное средство, Название, Инвентарный номер, МОЛ

Параметры:
docType = 'ОС'
zavod = 0
sklad — из колонки МОЛ (первое непустое значение, поддерживаются числовые и строковые форматы)

### Алгоритм обработки

1) Подключение по IMAP — сервис подключается к почтовому ящику, ищет непрочитанные письма
2) Извлечение вложений — для каждого письма извлекаются все вложения
3) Сохранение на диск — файл сохраняется с сгенерированным уникальным именем (osv_<timestamp>_<random>.xlsx)
4) Анализ структуры — определяется тип документа по наличию обязательных колонок:
- Сначала проверка на ОСВ
- Затем на ОС
- Если ни один тип не подходит — файл отклоняется
5) Сохранение в БД — для валидных файлов создаётся запись в таблице email_attachments
6) Уведомление — отправляется email отправителю с результатом (принят/отклонён)
7) SSE-событие — при успешном сохранении отправляется уведомление statementLoaded для обновления интерфейса

## Обработка ошибок

При ошибках анализа или сохранения:
- Файл удаляется с диска
- Отправитель получает письмо с указанием причины
- Ошибка логируется в БД и в консоль (для dev-режима)

## Управление вложениями

### Права доступа

Доступ к вложениям определяется через таблицу mol_access, которая связывает пользователя со складами (zavod, sklad). Метод getAttachmentsForUser(userId) возвращает только те вложения, у которых пара (zavod, sklad) присутствует в доступных пользователю складах.

### Удаление

Удаление выполняется с проверкой:
- Вложение существует
- Пользователь имеет доступ к складу вложения (если вложение привязано к складу)
- Физический файл удаляется с диска
- Запись удаляется из БД
- Отправляется SSE-уведомление statementDeleted

### Конфигурация

Настройки загружаются через ConfigModule из переменных окружения


## Сервисы

### ImapService
Отвечает за подключение к почтовому серверу по протоколу IMAP. Метод checkForNewEmails() открывает соединение, ищет непрочитанные письма, извлекает вложения и передаёт их в EmailProcessor. После обработки письма помечаются как прочитанные.

### SmtpService
Отправляет email-уведомления через SMTP. Используется для информирования отправителя о результате обработки его файла.

### EmailProcessor
Координирует полный цикл обработки одного вложения:
- Сохраняет файл на диск через EmailStorageService
- Запускает анализ через EmailFileAnalyzer
- При успехе сохраняет запись в БД и отправляет положительное уведомление
- При ошибке удаляет файл и отправляет отрицательное уведомление с причиной

### EmailFileAnalyzer
Анализирует структуру Excel-файла. Логика определения типа документа:
- Проверка наличия всех 6 колонок ОСВ → валидный ОСВ
- Проверка наличия всех 4 колонок ОС → валидный ОС
- Иначе → невалидный

Для ОСВ извлекает zavod (число) и sklad (первое непустое значение из колонки Склад)
Для ОС устанавливает zavod = 0, sklad — первое непустое значение из колонки МОЛ

### EmailAttachmentsService
Работа с записями в БД:
- getAttachmentsForUser(userId) — возвращает вложения с фильтрацией по доступным пользователю складам
- deleteAttachment(id, userId) — удаляет вложение с проверкой прав
При удалении также вызывает EmailStorageService.deleteFile() и отправляет SSE-уведомление.

### EmailStorageService
Управляет физическими файлами на диске:
- saveFile(filename, content) — сохраняет файл с уникальным именем, возвращает путь и имя
- deleteFile(filename) — удаляет файл
- fileExists(filename) — проверяет существование
- getFilePath(filename) — возвращает полный путь

### Логирование
Модуль использует стандартный Logger от @nestjs/common для записи в консоль. Основные события логируются:
- Подключение к IMAP
- Количество обработанных писем
- Сохранение/удаление файлов
- Результаты анализа (тип документа, склад)
- Ошибки обработки

Глобальные перехватчики и фильтры (настроенные в приложении) автоматически логируют все HTTP-запросы и необработанные исключения в таблицу logs. Для бизнес-событий (принятие/отклонение файлов) используется отдельный вызов LogsService.log() при необходимости детального аудита.
